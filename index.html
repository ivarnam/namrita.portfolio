About

The researcher

I am a PhD researcher at the Indian Institute of Technology Bombay, working at the intersection of formal methods and machine learning. My advisors are Prof. Ashutosh Gupta and Prof. Krishna S.

My work asks a deceptively simple question: can we mathematically prove that an AI system will behave as intended? I develop algorithms and tools that bring the rigor of formal verification to modern ML — particularly tree ensemble models like XGBoost and gradient-boosted trees used in high-stakes decisions.

Before my PhD, I spent 1.5 years as an Assistant System Engineer at Tata Consultancy Services, building backend services and real-time analytics dashboards for North American fleet-management clients on mainframe systems.

Research Tools

Projects

Four open-source tools built during my PhD — each translating a theoretical contribution into working code. Click any card to expand.

01 · ICLR 2026
Ensense

MILP + SMT-based sensitivity verification for XGBoost tree ensembles. Data-aware search using product-of-marginals objective with clause summaries for scalability.

GurobiZ3 XGBoostPython
// MILP encoding: sensitivity query
minimize Σᵢ w(xᵢ) · |f(x) − f(x′)|
subject to ‖x − x′‖∞ ≤ δ
solver: Gurobi (MILP) + Z3 (SMT)
objective: product-of-marginals P(x)

Addresses scalability bottleneck of prior verification methods by introducing clause summaries — compact representations of subtree constraints that reduce solver complexity without sacrificing soundness.

02 · Runtime Monitoring
RobQuant

Python runtime monitoring framework computing anytime lower/upper bounds on Lipschitz robustness scores over streaming input-output pairs using k-NN search.

Faissscikit-learnPython
// Lipschitz bound computation
L̂ = max_{x,x′ ∈ stream} |f(x) − f(x′)| / ‖x − x′‖
k-NN search: Faiss (approximate, scalable)
output: [L_lower, L_upper] anytime bounds

Enables real-time robustness monitoring without model access — works purely from observed input-output pairs. Anytime bounds tighten as more data streams in, giving both immediate and asymptotically precise robustness guarantees.

03 · Formal Verification
ViKriTi

Detects and formally characterises glitch regions — abrupt output oscillations — in gradient-boosted decision tree ensembles using MILP encoding.

GurobiXGBoostPython
// Glitch region detection
glitch(x) ≡ ∃ x′ ∈ Bδ(x): sign(f(x)) ≠ sign(f(x′))
AND ‖x − x′‖ < ε (arbitrarily small)
encoding: MILP via Gurobi

Glitches represent a fundamental failure mode — model predictions that flip unpredictably in tiny input regions. First formal study to characterise these in XGBoost and LightGBM, with algorithmic detection provably sound via MILP encoding.

04 · LTL Synthesis
Janak

Integrates LLMs with optimization-based LTL solvers — using LLM-generated candidates as seeds for fitness-aware search to synthesise temporal logic specs from demonstrations.

ChatGPT APILTL SolversPython
// LTL synthesis pipeline
input: demos τ₁…τₙ + NL explanation e
LLM: GPT → candidate φ̂ (seed)
search: fitness(φ) = consistency(φ, τ)
output: φ* s.t. ∀τᵢ: τᵢ ⊨ φ*

Bridges the gap between natural language intent and formal temporal specifications. LLM-generated seeds dramatically reduce the search space for LTL solvers, making specification synthesis from demonstrations practical for robot planning and autonomous systems.
